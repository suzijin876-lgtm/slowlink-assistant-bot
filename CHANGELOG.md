# 更新日志

本项目从V0.1.15起使用GitHub Releases保存安装包，Git仓库只保留源码、文档和发布记录。

## [0.1.19] - 2026-07-12

- 源频道只复制整条正文或图片说明完全为Telegram帖子链接的消息。
- 支持公开帖子链接`https://t.me/频道用户名/消息编号`和私密频道链接`https://t.me/c/频道ID/消息编号`。
- 纯文本、链接前后带说明文字、普通网址、邀请链接、Bot链接和频道置顶服务通知全部跳过。
- 被跳过的消息不会写入转发统计，也不会进入频道反应纠错状态；报表、权限、SQLite和watchdog逻辑保持不变。
- 增加严格链接过滤回归测试，覆盖正文、图片说明、公开链接、私密链接和无效链接。

## [0.1.18] - 2026-07-11

- 参考主SlowLink重写README，增加居中项目介绍、Release、GitHub Actions、Python、Docker和MIT License徽章。
- 将一键安装提前，并用简洁表格和流程图说明频道消息复制、主人私聊通知、定时报表与日常管理。
- 增加MIT License，完善敏感数据忽略规则，并从公开仓库移除内部规划稿。
- 增加可测试的发布构建脚本，统一生成app包、full包、update_log.txt和SHA256SUMS.txt四个本地归档文件。
- SHA256SUMS.txt只校验两个ZIP；GitHub Release只上传两个ZIP和校验文件，正文直接使用中文更新日志。
- 整理GitHub仓库简介、Topics、Wiki、Projects和历史Release展示；未修改Bot业务逻辑。

## [0.1.17] - 2026-07-11

- 公开脚本改成统一管理入口，无参数运行时提供安装、更新、卸载和退出菜单。
- 更新选项直接拉取GitHub最新稳定Release，并继续保留现有`.env`和`data/`。
- 卸载增加二级选择：可保留配置和数据库，也可选择彻底删除。
- 彻底删除继续要求输入`PURGE`，取消时不会先停止服务。
- 菜单统一从`/dev/tty`读取，兼容`curl | sudo bash`管道运行；原命令行参数继续保留给自动化。

## [0.1.16] - 2026-07-11

- 一键安装改成四步中文引导，明确说明每项配置的用途、来源和格式示例。
- Bot Token按用户选择改为明文输入，不再关闭终端回显。
- 报表群和源频道不再接受`1`这类正数ID，只接受负数聊天ID或Telegram用户名。
- 频道置顶操作产生的`pinned_message`服务通知直接跳过，不再尝试转发、写入纠错记录或计入异常。

## [0.1.15] - 2026-07-11

- 增加Ubuntu/Debian一键安装脚本`install.sh`，支持自动安装Docker、交互配置、指定版本、GitHub资产校验和保数据更新。
- 增加`manage.sh`，支持状态、日志、重启、更新、备份、卸载和受确认保护的彻底删除。
- 增加独立`uninstall.sh`一键卸载；默认保留配置和数据库，永久删除需要二次确认。
- 增加GitHub Actions自动发布：Tag触发测试、打包、SHA-256校验和Release创建。
- 仓库根目录不再保存`V0.1.x`文件夹和重复部署包，历史ZIP移至GitHub Releases。
- 所有历史版本说明合并到本文件。

## [0.1.14] - 2026-07-11

- 👎达到2个后进入60秒确认，低于2个自动取消，不再比较👍。
- 💩达到2个后跳过60秒等待并直接处理。
- 两种自动删除共用10分钟最多4条的批量保护。
- 纠错有效期改为1小时，👍不再参与判断或显示。
- 当前报告和运行状态增加纠错删除统计。
- SQLite增加`poop_count`幂等迁移，启动时检查频道反应设置。

## [0.1.13] - 2026-07-11

- 纠错状态和事件写入SQLite，并加入日报、周报和月报统计。

# 功能说明

当前版本：`0.1.19`

## 安装和更新

- `install.sh`支持Ubuntu和Debian一键安装。
- 自动安装Docker Engine和Docker Compose，下载GitHub最新稳定Release并校验SHA-256。
- 首次安装交互填写配置，Bot Token输入不回显，`.env`权限为`600`。
- 重复安装或`--update`保留`.env`、SQLite数据库和备份。
- `manage.sh`提供状态、日志、重启、更新、备份、卸载和彻底删除命令。
- `uninstall.sh`支持一条命令卸载；默认保留配置和数据库，`--purge`需输入`PURGE`确认。
- GitHub Tag会自动运行测试并创建带校验文件的Release。
- 首次安装使用四步中文引导，说明Token、主人ID、报表群和源频道分别填写什么。
- Bot Token按当前选择明文输入；报表群和源频道拒绝正数ID。
- 同一条公开命令提供安装、更新、卸载和退出菜单。
- 卸载菜单可选择保留配置和数据库，或通过`PURGE`二次确认彻底删除。

## 核心转发

- 监听 `.env` 中配置的 `SOURCE_CHANNEL_IDS`。
- 只接受整条正文或图片说明完全为Telegram帖子链接的消息。
- 支持`https://t.me/频道用户名/消息编号`和`https://t.me/c/频道ID/消息编号`。
- 纯文本、带额外文字的链接、普通网址、邀请链接、Bot链接和频道置顶通知直接跳过。
- 使用 Telegram Bot API 的 `copyMessage` 原样复制符合条件的源频道消息。
- 新消息只发送到 `OWNER_USER_ID` 对应的个人私聊。
- 只做帖子链接格式过滤，不做内容真实性识别、不做去重、不生成新的`t.me`链接。
- 只统计 Bot 自己的复制成功和失败，不判断内容真假。

## 报表

- 固定报表群由 `REPORT_CHAT_ID` 指定。
- 报表群不会收到源频道的新消息。
- 每天 `00:00` 发送昨日报。
- 每周一 `00:00` 发送上周报。
- 每月 1 日 `00:00` 发送上月报。
- 日报与周报或月报撞在同一分钟时，会自动合并成一条消息。
- 日报、周报、月报同时撞上时，也只发送和置顶一条组合消息。
- 组合消息发送成功后，各报表仍分别记录自己的发送状态。
- 报表发送后会尝试置顶。
- 新报表置顶成功后，会取消上一条报表置顶。
- 报表发送失败或置顶失败，会私聊提醒你。
- 日报显示日期、精确转发数量、活跃时段、首次转发和最后转发时间。
- 周报和月报显示统计周期、精确转发数量、日均数量、最活跃日期和最后转发时间。
- 空报表明确显示“转发：0条”和“运行状态：待命中”。
- 失败报表会显示最多 3 个失败原因摘要。
- 日报、周报、月报始终显示期间内的纠错删除数量。
- 主人保留和批量保护次数只有非零时才显示。
- 当前概览始终显示今日纠错删除数量。
- 运行状态始终显示今日纠错删除数量。

## 频道反应纠错

- 只处理配置在 `SOURCE_CHANNEL_IDS` 中、由 Bot 实时收到的新频道帖子。
- 频道建议只开启标准👎和💩反应；Telegram Bot API不能修改该设置，需要在频道设置中手动完成。
- `👎`不少于2个时进入待删除，等待60秒；期间降到2个以下会自动取消。
- `💩`不少于2个时跳过60秒倒计时，Bot收到反应统计后直接处理。
- 👍不参与判断，也不会显示在主人通知和运行日志中。
- 触发时私聊主人，并提供“保留”和“立即删除”按钮。
- 主人选择保留后，该帖子永久跳过自动删除。
- 10 分钟内最多自动删除 4 条，第 5 条触发批量保护并等待主人处理。
- 只处理1小时内的帖子，忽略自定义表情和功能启用前的旧帖子。
- Telegram可能延迟几分钟才发送频道反应统计；“直接处理”表示Bot收到统计后不再额外等待。
- 待删除状态、倒计时和处理结果保存到 SQLite，容器重启后不会丢失。
- 删除频道帖子不会删除主人私聊中的原消息副本。

## 命令

私聊可用命令：

```text
/help      查看命令
/status    查看运行状态
/report    查看当前概览
/recent    查看最近记录，可用 /recent 20
/check     自检
```

固定报表群可用命令：

```text
/report    查看当前概览
```

群里发送 `/start`、`/help`、`/id` 时，只会回复“此群已配置完成”，不会暴露群 ID。

## 权限和安全

- 私聊命令只允许 `OWNER_USER_ID` 使用。
- 固定报表群里只有你可以使用命令。
- 其他人私聊 Bot 会被忽略。
- Bot 被拉进非授权群组或频道，会自动退出。
- 同一个未授权群的退出提醒一小时最多发送一次，避免刷屏。
- `.env` 不应提交或打包到公开位置。

## 自检

`/check` 会检查：

- Bot 主循环是否能响应。
- SQLite 数据库是否可写。
- watchdog 状态文件是否近期更新。
- 源频道配置数量。
- 报表群权限是否正常。
- 今日转发和失败数量。
- 最近一次成功转发时间。

watchdog 状态含义：

- `守护：正常`：状态文件近期更新。
- `守护：异常`：状态文件不存在、读取失败或太久没更新。
- `守护：未知`：本地内存测试环境，没有真实状态文件。

## 数据和清理

- 使用 SQLite 保存转发记录、失败记录、报表发送状态和运行状态。
- 保存 Telegram update offset，重启后不会重复拉取旧更新。
- 保存频道纠错票数、待删除倒计时和处理事件。
- 自动清理 90 天前的转发记录。
- 每天自动备份 SQLite。
- 数据库备份保留最近 7 份。

## 稳定性

- Docker Compose 使用 `restart: always`。
- Docker healthcheck 会检查 Python 包和版本是否可加载。
- 消息堆积时主循环会连续处理更新，空闲时才短暂停顿。
- Telegram API 对临时网络错误、限流和 5xx 错误会做一次轻量重试。
- SQLite 报表查询字段已建索引，记录变多后统计更稳。
- 独立 systemd watchdog 监控 `slowlink_assistant_bot` 容器 CPU。
- CPU 连续 4 次超过 85%，每次间隔 20 秒，watchdog 会重启 Bot 容器。
- 重启后有 600 秒冷却，避免反复重启。
- watchdog 会写中文日志和状态文件。
- watchdog 自动重启 Bot 后，会私聊提醒你。

## 日志

- Bot Docker 日志为中文。
- Bot Docker 日志使用北京时间。
- 报表日志使用“日报、周报、月报”，不显示内部英文类型。
- 频道纠错日志记录待删除、取消、保留、删除失败和批量保护。
- watchdog 日志为中文。
- watchdog 状态文件路径：`/opt/slowlink_assistant_bot/data/watchdog_status.txt`。

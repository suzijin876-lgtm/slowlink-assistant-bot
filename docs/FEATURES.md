# 功能说明

当前版本：`0.1.33`

## 安装和更新

- `install.sh`支持Ubuntu和Debian一键安装。
- 自动安装Docker Engine和Docker Compose，下载GitHub最新稳定Release并校验SHA-256。
- 首次安装交互填写配置，Bot Token输入会显示，`.env`权限为`600`。
- 重复安装或`--update`保留`.env`、SQLite数据库和备份。
- `manage.sh`提供状态、日志、重启、更新、备份、卸载和彻底删除命令。
- `uninstall.sh`支持一条命令卸载；默认保留配置和数据库，`--purge`需输入`PURGE`确认。
- GitHub Tag会自动运行测试并创建带校验文件的Release。
- 首次安装使用六步中文引导，说明Token、主人ID、报表群、可选简报频道、源频道和可选SlowLink面板地址分别填写什么。
- Bot Token按当前选择明文输入；报表群、简报频道和源频道拒绝正数ID。
- 同一条公开命令提供安装、更新、卸载、修改配置和退出菜单。
- 修改配置支持每项回车保留原值，简报频道或SlowLink面板地址输入`0`可停用；只重建Bot容器，失败时自动恢复旧配置。

## 私聊控制面板

- 主面板顶部可显示`🌐SlowLink`跳转按钮，地址由服务器私有`.env`中的`SLOWLINK_PANEL_URL`配置。
- 未配置面板地址时不显示外链按钮；修改地址不需要改代码。
- 二级页面统一使用短“返回”按钮，封面管理使用紧凑两列布局。
- 卸载菜单可选择保留配置和数据库，或通过`PURGE`二次确认彻底删除。

## 核心转发

- 监听 `.env` 中配置的 `SOURCE_CHANNEL_IDS`。
- 只接受整条正文或图片说明完全为Telegram帖子链接的消息。
- 公开普通帖子和论坛话题支持`https://telegram.me/用户名/消息编号`与`https://telegram.me/用户名/话题编号/消息编号`。
- 私有普通帖子和论坛话题继续使用`https://t.me/c/内部ID/消息编号`与`https://t.me/c/内部ID/话题编号/消息编号`。
- 兼容旧公开`t.me`帖子输入，不改写用户原文中的链接。
- 纯文本、带额外文字的链接、普通网址、邀请链接、Bot链接和频道置顶通知直接跳过。
- 使用 Telegram Bot API 的 `copyMessage` 原样复制符合条件的源频道消息。
- 新消息只发送到 `OWNER_USER_ID` 对应的个人私聊。
- 只做帖子链接格式过滤，不做内容真实性识别、不做去重、不生成或改写帖子链接。
- 只统计 Bot 自己的复制成功和失败，不判断内容真假。

## 报表

- 固定报表群由 `REPORT_CHAT_ID` 指定。
- 可选简报频道由`REPORT_CHANNEL_ID`指定，留空时只向报表群发送。
- 报表群不会收到源频道的新消息。
- 每天 `00:00` 发送昨日报。
- 每周一 `00:00` 发送上周报。
- 每月 1 日 `00:00` 发送上月报。
- 报表标题分别显示“昨日日报”“上周周报”“上月月报”。
- 日报与周报或月报撞在同一分钟时，会自动合并成一条消息。
- 日报、周报、月报同时撞上时，也只发送和置顶一条组合消息。
- 主人可在Bot私聊中上传一张固定封面，日报、周报和月报将使用该图片并把动态统计显示在图片下方。
- 固定封面使用Telegram`file_id`复用，不需要图床；组合报表仍只发送一张图片和一份合并说明。
- 上传或替换封面后，Bot会在主人私聊发送使用当前真实数据的封面预览，不向报表群发送测试消息。
- 封面不可用或图片说明过长时自动改发纯文字，不影响报表发送状态与置顶。
- 组合消息发送成功后，各报表仍分别记录自己的发送状态。
- 报表群和简报频道分别发送、置顶并取消各自上一条简报置顶。
- 两个目标分别记录投递状态；某一边失败时只重试失败目标，不重复发送成功目标。
- 主人可在私聊按钮面板中分别开关群和频道的日报、周报、月报，设置写入SQLite。
- 同日触发多个周期时，每个目标只合并自身启用的报表。
- 报表发送失败或置顶失败，会私聊提醒你。
- 群内简报使用有效转发数量，按源帖排除已记录删除的帖子，删除发生在次日时仍从原帖所属周期扣除。
- 日报显示日期、有效转发、较前日变化、高峰时段、高峰转发、首次转发和最后转发时间。
- 周报和月报显示周期、有效转发、较前一周期变化、日均、活跃天数、最活跃日、首次转发和最后转发时间。
- 群内当前概览显示日期、有效转发、昨日同期变化、高峰时段、高峰转发、首次转发和最后转发时间。
- Bot在SQLite中保存统计覆盖开始时间；历史周期没有完整覆盖时显示“暂无可比数据”，不会将缺失记录当成零。
- 首个不完整周或月份会显示实际数据范围，并按实际覆盖天数计算日均；完整周期不增加这行说明。
- 每次正式报表发送成功后保存SQLite快照，记录周期、统计数量、比较区间、显示正文和Telegram消息ID。
- 群内简报不显示纠错删除、运行状态或异常记录；详细错误保留在SQLite和服务器日志中。
- 主人私聊按钮中的运行状态和当前报告显示原始转发、纠错删除与异常信息，系统自检显示运行检查结果。

## 频道反应纠错

- 只处理配置在 `SOURCE_CHANNEL_IDS` 中、由 Bot 实时收到的新频道帖子。
- 频道建议只开启标准👎和💩反应；Telegram Bot API不能修改该设置，需要在频道设置中手动完成。
- `👎`不少于2个时进入待删除，等待60秒；期间降到2个以下会自动取消。
- `💩`不少于2个时跳过60秒倒计时，Bot收到反应统计后直接处理。
- 👍不参与判断，也不会显示在主人通知和运行日志中。
- 触发时私聊主人，并提供“保留”和“立即删除”按钮。
- 主人选择保留后，该帖子永久跳过自动删除。
- 10 分钟内最多自动删除 4 条，第 5 条触发批量保护并等待主人处理。
- 只处理1小时内的帖子，忽略自定义表情和功能启用前的旧帖子。
- Telegram可能延迟几分钟才发送频道反应统计；“直接处理”表示Bot收到统计后不再额外等待。
- 待删除状态、倒计时和处理结果保存到 SQLite，容器重启后不会丢失。
- 删除频道帖子不会删除主人私聊中的原消息副本。
- 主人在私聊中回复Bot复制消息`删`或`删除`，会立即删除对应源帖并记入纠错统计。
- 主人回复`已删除`，会记录已经手动删除的源帖，不再次调用Telegram删除接口。

## 按钮面板

- Telegram命令列表只显示`/start`，用于打开主人私聊主面板。
- 主面板显示版本、模式、运行状态和操作提示，以三行两列宽版按钮集中进入六项功能。
- 主面板包含当前报告、运行状态、最近记录、系统自检、简报设置和封面管理。
- 详情页面使用同一条消息切换，提供刷新和返回主面板按钮，减少私聊刷屏。
- 简报设置分别显示报表群和简报频道的日报、周报、月报开关。
- 封面管理支持更换、预览和停用；点击更换后直接发送下一张图片，无需填写图片说明。
- 原`/status`、`/report`、`/recent`、`/check`、`/cover`等入口继续隐藏兼容。
- 固定封面只用于自动发送的日报、周报和月报，当前概览保持纯文字。

回复Bot转发的私聊消息：

```text
删         立即删除源频道帖子
删除       立即删除源频道帖子
已删除     记录源帖已被主人手动删除
```

- 固定报表群只显示主人可用的“当前报告”按钮。
- 群里使用旧`/start`、`/help`、`/id`入口时，只回复配置完成面板，不暴露群ID。

## 权限和安全

- 私聊面板和隐藏兼容命令只允许`OWNER_USER_ID`使用。
- 固定报表群里只有主人可以操作按钮。
- 其他人私聊 Bot 会被忽略。
- Bot 被拉进非授权群组或频道，会自动退出。
- 同一个未授权群的退出提醒一小时最多发送一次，避免刷屏。
- `.env` 不应提交或打包到公开位置。

## 自检

“系统自检”会检查：

- Bot 主循环是否能响应。
- SQLite 数据库是否可写。
- watchdog 状态文件是否近期更新。
- 源频道配置数量。
- 报表群权限是否正常。
- 简报频道是否已配置，以及Bot是否具备发帖和置顶权限。
- 今日转发和失败数量。
- 最近一次成功转发时间。

watchdog 状态含义：

- `守护：正常`：状态文件近期更新。
- `守护：异常`：状态文件不存在、读取失败或太久没更新。
- `守护：未知`：本地内存测试环境，没有真实状态文件。

## 数据和清理

- 使用SQLite保存转发记录、失败记录、报表发送状态、每目标投递状态、简报开关和运行状态。
- 保存统计覆盖开始时间和已发送报表快照，用于区分真实零转发与历史数据缺失，并追溯当时显示内容。
- 保存 Telegram update offset，重启后不会重复拉取旧更新。
- 保存频道纠错票数、待删除倒计时和处理事件。
- 自动清理 90 天前的转发记录。
- 每天自动备份 SQLite。
- 数据库备份保留最近 7 份。

## 稳定性

- Docker Compose 使用 `restart: always`。
- Docker healthcheck 会检查 Bot 主循环心跳是否在 120 秒内更新。
- 消息堆积时主循环会连续处理更新，空闲时才短暂停顿。
- Telegram API 对临时网络错误、读取超时、限流和 5xx 错误会做一次轻量重试。
- 长轮询连续超时只写一行中文警告；未知程序异常仍记录完整堆栈。
- SQLite 报表查询字段已建索引，记录变多后统计更稳。
- 独立 systemd watchdog 监控 `slowlink_assistant_bot` 容器 CPU 和主循环心跳。
- CPU 连续 4 次超过 85%，每次间隔 20 秒，watchdog 会重启 Bot 容器。
- 心跳连续 2 次超过 120 秒或不存在时，watchdog 会重启 Bot 容器。
- 重启后有 600 秒冷却，避免反复重启。
- watchdog 会写中文日志和状态文件。
- watchdog 自动重启 Bot 后，会私聊提醒你。

## 日志

- Bot Docker 日志为中文。
- Bot Docker 日志使用北京时间。
- Docker JSON 日志单文件上限 10MB，最多保留 3 份。
- 报表日志使用“日报、周报、月报”，不显示内部英文类型。
- 频道纠错日志记录待删除、取消、保留、删除失败和批量保护。
- watchdog 日志为中文。
- watchdog 状态文件路径：`/opt/slowlink_assistant_bot/data/watchdog_status.txt`。
